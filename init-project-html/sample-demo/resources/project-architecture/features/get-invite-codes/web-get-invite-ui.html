web-get-invite-ui

自身職責：登入後顯示「我的邀請碼」、列表載入、生成按鈕。本頁只談自身函式 I/O 與內部資料流。

函式 I/O

函式簽名副作用用途
useInviteCodes() in: —
out: {list, isLoading, refresh} 		network GET 已發放碼 查詢鍵／快取列表。
createInviteCode in: —
out: Result<NewCode, UiError> 		network POST 把伺服器產生的新碼樂觀 prepend 到列表。
copyCodeToClipboard in: code: string
out: void 		browser clipboard API 使用者點復制。
formatQuotaError in: RetryAfter | null
out: UiMessage 		pure 把 429 訊息翻成本地文字。

變數與業務用途

本子模組以前端組件狀態為主；下表為決定使用者體驗的識別子。

變數型別作用域業務用途
list InviteCode[] instance 使用者已發放的邀請碼清單；驅動畫面呈現與「複製」按鈕可用性；保留順序讓使用者能識別最近一次取得的碼。
isLoading boolean instance 生成 API 進行中旗標；防止連按重複生成造成額度被無意義消耗。
retryAfter number seconds | null instance 後端 429 帶回的下次可生成時間；按鈕禁用倒數的依據，業務上提示使用者「節流而非系統故障」。
Result {ok: NewCode} | {err: UiError} call 一次「生成」呼叫結果；驅動是否要把新碼樂觀插入 list，或顯示錯誤 banner。
UiMessage {lang_tag, text} call 額度／網路錯誤的本地化呈現；使用者透過此訊息知道是自己的額度問題還是系統暫時不可用。

內部資料流

  1. 頁面掛載 → useInviteCodes 取得 list 顯示。
  2. 使用者點生成 → createInviteCode：UI idle → loading。
  3. Result.ok → 把 NewCode prepend 到 list（保持其餘元素穩定）。
  4. Result.err 為 quota → formatQuotaError 顯示 UiMessage，list 不變。
  5. 使用者選擇 copyCodeToClipboard(code)，不影響資料模型。

本子模組可暴露的錯誤

來源UI 行為
未登入（401）路由守衛攔截 → 導向登入；本子模組不顯示業務錯誤。
quota（429）formatQuotaError banner；按鈕短暫禁用至 retryAfter。
網路錯誤泛用提示；可重試。
