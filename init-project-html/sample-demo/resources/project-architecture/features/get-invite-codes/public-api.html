Submodule · public-api

public-api（iss）

自身職責：HTTP 邊界：認證解析、輸入綁定、把網域結果映射為狀態碼。

函式 I/O

函式簽名副作用用途
AuthMiddleware in: http.Request
out: Principal | 401 		io 解 cookie / JWT 驗證並注入 Principal.UserID。
CreateInviteCodeHandler in: http.Request, Principal
out: http.Response 		io 寫 body 路由綁定的對外入口。
mapDomainError in: error
out: (httpStatus, ErrorBody) 		pure quota → 429（含 Retry-After）等對映。
writeCreated in: NewCode, http.ResponseWriter
out: — 		io 寫 201 body 序列化新碼 + metadata。

變數與業務用途

HTTP 邊界子模組持有的識別子；本子模組的責任是「正確識別調用者」與「正確翻譯網域結果為協議語意」。

變數型別作用域業務用途
http.Request request call 邊界輸入；只在 handler 期間有效。
Principal.UserID UUID call 由 middleware 注入；業務上代表「為誰生成邀請碼」；禁止從 body 取以避免使用者冒名為他人發碼。
httpStatus int call mapDomainError 的輸出之一；ErrQuotaExceeded 必須對映成 429 才能搭配 Retry-After 讓 UI 知道是節流。
ErrorBody {code, message} call 結構化錯誤體；code 給 UI 對映本地化文字（例如「今日額度用盡」）。
Retry-After int seconds call 節流提示 header；UI 透過它倒數禁用按鈕，避免使用者反覆 429。
NewCode string call 網域回傳的新碼；序列化為 201 body 給使用者複製分享。
http.ResponseWriter writer call 對外副作用通道；保證一次回應只寫一次。

內部資料流

  1. Request → AuthMiddleware → 注入 Principal{UserID}，失敗即 401。
  2. Principal.UserID 作為網域層輸入；本子模組不持有 token 明文。
  3. 從網域層取得 NewCode | error。
  4. NewCode → writeCreated；error → mapDomainError。
