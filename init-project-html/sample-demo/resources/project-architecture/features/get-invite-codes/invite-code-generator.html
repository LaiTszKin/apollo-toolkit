invite-code-generator

自身職責：把熵編碼成符合產品規格的可讀字串。無 I/O 副作用（除讀取隨機源）。本頁只列自身函式 I/O 與內部資料流。

函式 I/O

函式簽名副作用用途
NewGenerator in: Config{bits, alphabet, groupSize}
out: Generator 		pure 建構，驗證 bits >= 64、字元表非空且不含易混字。
Generator.Next in: —
out: string | ErrEntropy 		io 讀 crypto/rand 單次抽碼。
encode in: bytes, alphabet
out: string 		pure 把位元組以 Crockford Base32（或同類）映射成字元。
group in: string, groupSize
out: string 		pure 每 N 位插入 -，便於人讀。

變數與業務用途

本子模組屬於純函式工具；下表分兩段：上半為建構時固定的設定，下半為單次 Next 期間的臨時值。

變數型別作用域業務用途
Config {bits, alphabet, groupSize} instance 整個 Generator 生命週期固定；決定碼強度（bits）、可讀性（alphabet）、人讀分組（groupSize）。
bits int instance 熵預算；決定隨機字串對 UNIQUE 碰撞的容忍度；過低會推高呼叫端的重試次數與失敗率。
alphabet string instance 字元表；剃除 0/O、1/I/l 等易混字，業務上降低使用者抄錄錯誤率，間接提高分享成功率。
groupSize int instance 每幾個字元插入連字號；只影響顯示／輸入體驗，不影響熵或唯一性。
bytes / buf []byte call 從 crypto/rand 讀出的原始熵；只在單次 Next 期間活著，不持久化、不外傳，避免被推測下一次值。
raw string call 經 base32 編碼後尚未分組的中介字串；只是內部步驟，不對外。
out / candidate string call 最終回傳的候選碼；本子模組不快取、不記錄；唯一性由呼叫端結合 DB UNIQUE 保證。

內部資料流（單次 Next）

  1. 從 crypto/rand 讀 bytes[ceil(bits/8)]。
  2. bytes → encode(bytes, alphabet) → raw: string。
  3. raw → group(raw, groupSize) → 最終字串。
  4. 內部不持有狀態（除設定）；多次呼叫互相獨立，無快取無記錄。

本子模組會拋出的錯誤

錯誤條件
ErrEntropycrypto/rand.Read 短讀／失敗（罕見）。
ErrConfigNewGenerator 收到非法 Config。

注意：「唯一性」不在本子模組；DB UNIQUE 違例由呼叫端處理（重新呼叫 Next）。
