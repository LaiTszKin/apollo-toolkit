registration-service

自身職責：在單一交易內以邀請碼狀態判定是否建立 user 並核銷邀請碼。本頁只記述本子模組自身函式的 I/O 與內部資料流；與其他子模組之間的呼叫關係請見宏觀架構圖。

函式 I/O

函式簽名／I/O副作用用途
RegisterWithInvite in: ctx, RegisterInput{Email, Password, InviteCode}
out: NewUserID | error 		tx 開／關交易
db 條件成立時寫入 user + 標記邀請碼已用
hash 對密碼做 hash 		對外唯一入口。
validateInput in: RegisterInput
out: NormalizedInput | ErrInvalidInput 		pure email 規範化、邀請碼 trim／大小寫策略、密碼最小強度。
decideInviteValid in: InviteRow | nil, now
out: true | ErrInviteInvalid 		pure 判斷邀請列是否存在、未過期、未核銷、allowed_registration==true。
hashPassword in: plaintext
out: passwordHash 		cpu 高成本（bcrypt/argon2） 密碼絕不外傳明文；本函式回傳後 plaintext 應被丟棄。

變數與業務用途

下列為本子模組在「邀請碼註冊」業務裡實際持有／流動的識別子。型別僅供讀者對齊，業務用途優先。

變數型別作用域業務用途
ctx context.Context call 限制單次註冊請求佔用資源的時間預算；逾時或取消即放棄交易，避免長時間握列鎖把其他註冊請求拖垮。
RegisterInput {Email, Password, InviteCode} call 使用者註冊意圖承載：要為誰建立帳號、用什麼憑證、套用哪張邀請碼；任一欄缺值即無法進入業務判定。
NormalizedInput {email_lc, code_norm, pwd_plain} call 規範化後的業務輸入；保證 email／邀請碼比較在同一基準上，避免大小寫造成的可避免失敗。pwd_plain 在 hash 之後即被丟棄。
row / InviteRow DB row tx 「能否註冊」的唯一證據；AllowedRegistration / ConsumedAt / ExpiresAt 三欄直接決定進入「拒絕」或「建立」分支；列鎖在本 tx 內持續至結束。
now time.Time call 決策基準時間：判斷邀請碼是否過期、寫入 consumed_at 的官方時戳；測試可注入以避免依賴牆鐘。
passwordHash []byte call 不可逆密碼承載；確保任何明文不會離開本子模組或進入持久化路徑與日誌。
UserToInsert {email_lc, passwordHash, invite_id} tx 把「業務通過」結果翻譯成 DB 寫入承載；invite_id 把新使用者與其邀請來源綁定，作為日後審計依據。
tx *sql.Tx tx 把「邀請判定 + 建立 user + 核銷邀請」綁成原子單元；commit／rollback 決定本次註冊是否在世界上留下任何持久化痕跡。

內部資料流

下列步序描述同一個 RegisterWithInvite 呼叫內部資料如何在自身函式之間流動。每一步只標記值的變化與作用，不展開外部子模組的細節。

  1. RegisterInput → validateInput → NormalizedInput{email_lc, code_norm, pwd_plain}。
  2. NormalizedInput.code_norm 作為查鎖讀的鍵；讀回 InviteRow | nil，記在區域變數 row。
  3. row 連同 time.Now() 傳入 decideInviteValid；不通過則拋 ErrInviteInvalid 並結束（交易在外層被回滾）。
  4. NormalizedInput.pwd_plain → hashPassword → passwordHash；之後 pwd_plain 不再使用。
  5. 組合 UserToInsert{email_lc, passwordHash, invite_id=row.ID}，作為寫入承載。
  6. 條件分支結束後將控制權交給交易管理；本子模組函式自此不再持有任何明文敏感資料。

本子模組會拋出的錯誤

錯誤觸發條件後續責任
ErrInvalidInputvalidateInput 失敗呼叫端決定 HTTP 對映；本子模組不開交易。
ErrInviteInvaliddecideInviteValid 為 false呼叫端負責回滾交易並回應對外錯誤碼。
ErrTransient外部 I/O 暫時失敗（例如 hash CPU 中斷、context 取消）呼叫端決定是否重試；本子模組僅保證未提交。

回滾／重試怎麼影響其他子模組的狀態，屬於跨界議題；請見宏觀架構圖的 manifest 與資料行邊。
