Submodule · public-api

public-api（reg）

自身職責：HTTP 邊界。處理綁定、輸入 schema 檢查、把網域結果映射為狀態碼／錯誤 payload。

函式 I/O

函式簽名副作用用途
RegisterHandler in: http.Request
out: http.Response 		io 讀／寫 socket 路由綁定的對外入口。
decodeRegisterInput in: io.Reader
out: RegisterInput | ErrBadRequest 		io 讀 body JSON 解碼 + 基本 schema 校驗。
mapDomainError in: error
out: (httpStatus, ErrorBody) 		pure 網域 error → HTTP 對映表。
writeSuccess in: NewUserID, http.ResponseWriter
out: — 		io 寫 201 body 序列化 user id + 必要 header。

變數與業務用途

HTTP 邊界子模組持有的識別子，幾乎全在單次 handler 呼叫中存活。

變數型別作用域業務用途
http.Request request call 邊界輸入；只在 handler 期間有效，handler 結束後資源歸還，不可在背景任務繼續使用。
RegisterInput {Email, Password, InviteCode} call HTTP body 解碼後的網域層承載；schema 校驗通過代表「結構面可進入業務」，避免錯誤輸入觸發無謂的 DB 開銷。
httpStatus int call mapDomainError 的輸出之一；業務面決定協議語意（409 與 503 不可混用，前者要使用者改、後者要使用者稍後再試）。
ErrorBody {code, message} call 結構化錯誤體；code 給 UI 對映本地化文字，message 僅供開發者排錯。
http.ResponseWriter writer call 對外副作用通道；本子模組保證一次回應只寫一次，避免重複寫入造成協議違例與使用者看到的不一致結果。

內部資料流

  1. http.Request.Body → decodeRegisterInput → RegisterInput（失敗 → ErrBadRequest）。
  2. RegisterInput 作為傳入網域層的參數；本子模組不持有交易、密碼明文也只在這一函式內存在。
  3. 從網域層拿到 NewUserID | error：
    • NewUserID → writeSuccess。
    • error → mapDomainError → 對應 HTTP 狀態 + JSON。
  4. 處理結束 / response 完成寫入後生命週期終止。

本子模組會回應／轉化的錯誤

條件HTTP
schema 失敗400 BAD_REQUEST
網域回 ErrInviteInvalid409 INVITE_INVALID
網域回 ErrTransient503 SERVICE_UNAVAILABLE
panic 或未分類500 INTERNAL
