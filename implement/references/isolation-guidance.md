# 隔離環境使用指引

判斷是否需要在獨立 worktree 中進行實作。

## 需要隔離（使用 worktree）

以下情況建議使用隔離路徑：

- 變更涉及 **5 個以上檔案**
- 變更涉及 **跨模組的修改**（如新增 API 端點 + 資料庫 migration + 前端頁面）
- **實驗性或高不確定性**的修改，不希望影響當前工作目錄
- 需要在實作過程中頻繁切換上下文
- spec 指定了分支命名規則

## 不需要隔離（快速路徑）

以下情況直接在當前分支實作即可：

- **單一或少數檔案**的變更（1-3 個檔案）
- 變更範圍**完全集中在單一模組**內
- **純粹的 bug fix**，不涉及新功能
- 用戶明確要求不開新分支

## 快速判斷

```
變更檔案 > 5 或跨模組？
  ├── 是 → 使用隔離路徑（worktree）
  └── 否 → 當前分支實作即可
```
